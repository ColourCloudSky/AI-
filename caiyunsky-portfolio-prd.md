## 一、产品概述

### 1.1 产品定位

彩云译设计作品集网站是一个面向潜在雇主和客户的个人设计能力展示平台。它不是内容社区，不是博客站点，而是一张可交互的高级职业名片——用作品建立专业信任，用内容深度强化行业认知，最终将访客转化为私域联系人。

### 1.2 核心价值主张

对访客（雇主/客户）而言：在一个站点内快速、完整地了解设计师的能力边界、设计思维深度和过往项目经验，降低筛选和沟通成本。

对站长（彩云sky）而言：拥有一个不依赖第三方平台算法的独立展示窗口，所有流量最终沉淀到微信和公众号私域。

### 1.3 竞品参考与差异点

| 维度 | Behance | Dribbble | 本产品 |
|------|---------|----------|--------|
| 定位 | 项目级深度展示 | 视觉快照展示 | 个人能力全景展示 |
| 内容深度 | 长图文叙事 | 单图/短视频 | 中等——图为主+简要上下文 |
| 社交功能 | 点赞/评论/关注 | 点赞/评论/招聘 | 无社交，纯展示+转化 |
| 内容管理 | 平台内编辑 | 平台内编辑 | Notion/语雀作为CMS |
| 转化目标 | 平台内社交关系 | 平台内社交关系 | 站外私域（微信/公众号） |

## 二、用户与场景

### 2.1 目标用户画像

**主要用户：潜在雇主/客户（占访问量预估70%）**

- 设计团队负责人、HR、猎头，正在筛选候选人
- 需要外包设计服务的产品负责人或创业者
- 典型行为：通过链接或搜索引擎到达，快速浏览作品判断能力匹配度，决定是否发起联系
- 停留时间预期：3-8分钟

**次要用户：设计行业同行（占访问量预估30%）**

- 通过公众号文章、朋友圈分享等渠道到达
- 典型行为：看作品找灵感，顺便看看有没有可下载的资源
- 可能转化为公众号粉丝

### 2.2 核心使用场景

| 场景 | 用户动机 | 期望路径 | 成功标准 |
|------|---------|---------|---------|
| 雇主筛选候选人 | 判断设计能力是否匹配岗位 | 首页→能力分类→案例详情→联系我 | 发起微信添加或邮件联系 |
| 客户寻找设计外包 | 确认设计师能做什么、做得怎么样 | 首页→作品浏览→关于我→联系我 | 发起合作咨询 |
| 同行浏览学习 | 看作品找灵感、下载资源 | 首页→作品/资源→关注公众号解锁 | 关注公众号 |

## 四、功能需求详述

### 4.1 首页

**目标：** 5秒内传达"这是谁、能做什么、作品水平如何"，引导访客深入浏览。

**内容组成：**

- Hero区域：个人身份标识（名称、一句话定位、头像/品牌图形），配合一张高质量视觉主图或动效作为背景
- 能力标签导航：UI设计 / 动效设计 / 3D设计 / 品牌设计，点击跳转对应作品分类
- 精选作品网格：从各分类中各选1-2个代表作，以封面图+项目名称展示，点击进入详情
- 快捷入口：设计方法论、设计资源、关于我
- 底部CTA：联系我

**交互要点：**

- 首屏加载完成时间控制在2秒以内
- 精选作品区支持hover预览效果（PC端）
- 移动端采用单列布局，精选作品纵向排列

### 4.2 作品集模块

**目标：** 按能力维度系统化展示设计作品，让访客快速判断特定领域的设计能力。

**4.2.1 作品列表页**

- 顶部tab切换：UI设计 / 动效设计 / 3D设计 / 品牌设计
- 每个分类下以卡片网格展示案例缩略图
- 卡片信息：封面图 + 项目名称 + 一句话描述（选填）
- PC端每行3-4张卡片，移动端每行1-2张
- 点击卡片进入案例详情页

**4.2.2 案例详情页**

每个案例详情页包含以下内容模块（部分选填）：

| 模块 | 必填/选填 | 说明 |
|------|----------|------|
| 项目名称 | 必填 | 案例标题 |
| 项目背景 | 选填 | 2-3句话说明项目是什么、解决什么问题（建议填写，对雇主说服力关键） |
| 我的角色 | 选填 | 在项目中承担的具体职责 |
| 作品图片 | 必填 | 支持多图，纵向长图文排列，支持点击放大查看 |
| 视频 | 选填 | 动效演示、交互演示等辅助视频，支持内嵌播放 |
| 设计思路 | 选填 | 简要说明设计决策背后的思考 |
| 项目成果 | 选填 | 数据指标、上线效果等 |

**关于"选填"字段的产品建议：**
虽然标注为选填以降低内容录入门槛，但强烈建议至少为每个案例填写"项目背景"。原因：核心受众是雇主/客户，他们需要的不仅是"这个人画面好看"，而是"这个人能理解业务、能解决问题"。一段简短的项目上下文（3-5行即可）能让作品的说服力提升一个量级。

**交互要点：**

- 图片支持懒加载，确保长页面滚动流畅
- 图片支持lightbox全屏查看
- 视频采用内嵌播放器，默认静音，支持全屏
- 详情页底部展示"更多作品"推荐（同分类下的其他案例）
- 返回列表时保持之前的滚动位置和tab状态

### 4.3 设计方法论模块

**目标：** 通过精选的设计思维类文章，增强访客对设计师专业深度的认知。

**内容策略：**

不做全量同步。从公众号历史文章中精选与"设计体系、设计方法论、设计思维"直接相关的内容，排除纯工具评测和行业资讯类文章。建议首批精选8-15篇。

**展示方式：**

- 文章列表：封面图 + 标题 + 摘要（1-2句） + 发布日期
- 点击行为：跳转到公众号原文（微信内打开）或在站内展示全文（二选一，见下方讨论）
- 列表按时间倒序排列

**站内阅读 vs 跳转公众号的权衡：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 跳转公众号原文 | 零维护成本，天然引流公众号 | 体验割裂，PC端打开微信文章不友好 |
| 站内展示全文 | 体验流畅，阅读数据可控 | 需要内容同步机制，维护成本增加 |
| 推荐方案：站内展示 + 底部引导关注公众号 | 兼顾体验和转化 | 需要建立内容同步流程 |

### 4.4 设计资源模块

**目标：** 用免费资源作为钩子，将访客转化为公众号粉丝。

**资源类型：** 设计模板、Figma插件、设计源文件、工具合集等。

**展示方式：**

- 资源卡片网格：封面图 + 资源名称 + 一句话描述 + 资源类型标签
- 下载流程：点击下载 → 弹出公众号二维码 → 提示关注后回复关键词获取下载链接
- 每个资源标注文件格式和大小

**交互要点：**

- 未解锁状态下，资源封面图正常展示，下载按钮显示"关注公众号解锁"
- 解锁引导弹窗包含：公众号二维码 + 关键词文案 + 简要操作步骤说明
- 资源数量初期不宜过多，5-10个高质量资源即可

### 4.5 关于我

**目标：** 提供个人背景信息，建立信任，引导联系。

**内容组成：**

- 个人照片或品牌化头像
- 职业身份描述（一段话：谁、做什么、擅长什么、做了多久）
- 职业经历时间线（关键节点：公司/职位/年份，不需要详细描述）
- 核心能力标签（可视化呈现）
- 社交/媒体链接：公众号、知识星球等
- 联系方式区域（主CTA）

### 4.6 全局元素

**4.6.1 导航栏**

- 内容：Logo/名称 + 作品集 + 方法论 + 资源 + 关于我 + 联系我（高亮按钮）
- PC端：顶部固定导航，滚动时半透明毛玻璃效果
- 移动端：汉堡菜单，"联系我"按钮始终可见

**4.6.2 联系我（全局CTA）**

- 这是整个网站的北极星动作入口
- PC端：导航栏右侧固定按钮 + 页面底部大面积CTA区域
- 移动端：底部固定悬浮按钮
- 点击后展示：微信二维码 + 邮箱地址 + 一句引导语（"期待与你交流设计"）
- 不使用联系表单（直接给微信更高效）

**4.6.3 页脚**

- 版权声明
- 公众号二维码（小尺寸）
- 备案号（如果需要）

## 五、内容管理方案

### 5.1 CMS选型

采用Notion作为内容管理后台（语雀作为备选），通过Notion API读取内容数据渲染到前端。

**Notion数据库结构建议：**

| 数据库 | 字段 |
|--------|------|
| 作品库 | 项目名称、分类（UI/动效/3D/品牌）、封面图、项目背景、我的角色、设计思路、项目成果、作品图片（多图）、视频链接、排序权重、是否精选、发布状态 |
| 文章库 | 标题、封面图、摘要、正文内容/公众号原文链接、发布日期、发布状态 |
| 资源库 | 资源名称、封面图、描述、文件格式、文件大小、下载关键词、资源类型标签、发布状态 |

**内容更新流程：**

1. 在Notion中新建或编辑条目
2. 将"发布状态"改为"已发布"
3. 网站通过API自动拉取最新数据（可设置定时刷新或手动触发）

### 5.2 图片和视频存储

- 图片：建议使用图床服务（如阿里云OSS、Cloudflare R2）而非直接依赖Notion的图片链接（Notion图片链接有过期风险）
- 视频：上传至第三方视频平台（如Bilibili、YouTube）后以embed方式嵌入，或使用云存储直链

## 六、转化策略

### 6.1 转化漏斗设计

```
访客到达 → 浏览作品（建立信任）→ 深入了解（方法论/关于我）→ 转化行为
                                                                    ├── 主转化：添加微信（雇主/客户场景）
                                                                    └── 副转化：关注公众号（同行/资源场景）
```

### 6.2 CTA布局策略

| 位置 | CTA类型 | 触发场景 |
|------|---------|---------|
| 导航栏右侧 | "联系我"按钮 | 全局常驻，任何时刻可触达 |
| 案例详情页底部 | "对这类项目感兴趣？聊聊" | 看完具体案例后的转化窗口 |
| 关于我页面底部 | 微信二维码 + 邮箱 | 了解完个人背景后的自然转化点 |
| 资源下载弹窗 | 公众号二维码 | 资源驱动的公众号关注 |
| 移动端底部悬浮 | "联系我"按钮 | 全局常驻 |

### 6.3 微信添加引导设计

- 展示个人微信二维码（非企业微信）
- 附带一句引导语，避免生硬（如"扫码加我微信，聊聊设计"）
- 二维码下方附上邮箱作为备选联系方式
- 不使用表单提交，减少转化摩擦

## 七、技术方案建议

### 7.1 技术选型

| 层面 | 推荐方案 | 理由 |
|------|---------|------|
| 框架 | Next.js（SSG/ISR） | SEO友好，静态生成性能好，Notion API集成成熟 |
| 部署 | Vercel / Cloudflare Pages | 免费额度足够个人站点，全球CDN |
| CMS | Notion API | 免费，操作门槛低，已有使用习惯 |
| 图片存储 | Cloudflare R2 / 阿里云OSS | 稳定，避免Notion图片链接过期 |
| 视频嵌入 | Bilibili iframe / YouTube embed | 零存储成本，播放器成熟 |
| 域名 | 自定义域名（如caiyunsky.design） | 专业感，SEO友好 |
| 统计 | Google Analytics / 百度统计 | 追踪访客行为和转化数据 |

### 7.2 性能要求

- 首页首屏加载时间 ≤ 2秒（3G网络下 ≤ 4秒）
- 图片全部使用WebP格式，支持响应式尺寸
- Lighthouse Performance评分 ≥ 90
- 全站支持响应式，断点：移动端（<768px）、平板（768-1024px）、PC端（>1024px）

### 7.3 SEO基础要求

- 每个页面独立的title和description
- 语义化HTML结构
- Open Graph标签（分享到社交媒体时的预览卡片）
- sitemap.xml自动生成
- robots.txt配置

## 九、设计方向

### 9.1 视觉调性

延续"彩云译设计"公众号的品牌调性，保持视觉一致性。

**关键原则：**

- 内容优先：视觉设计为作品服务，不喧宾夺主。大面积留白，让作品图片成为页面主角
- 专业克制：不用花哨动效、不堆装饰元素。用排版和空间感传递高级感
- 品牌延续：色彩、字体、图形语言与公众号保持统一

### 9.2 响应式策略

- PC端（>1024px）：作品网格3-4列，充分利用大屏展示作品细节
- 平板端（768-1024px）：作品网格2列
- 移动端（<768px）：作品网格1列，导航收起为汉堡菜单，联系CTA底部悬浮

## 十、数据指标

### 10.1 北极星指标

**微信添加转化率** = 点击"联系我"并展示二维码的UV / 全站UV

### 10.2 辅助指标

| 指标 | 定义 | 目标值（上线3个月） |
|------|------|-------------------|
| 月访问UV | 独立访客数 | ≥ 500 |
| 平均浏览深度 | 人均浏览页面数 | ≥ 3页 |
| 案例详情页到达率 | 进入至少一个案例详情页的UV占比 | ≥ 40% |
| 联系我点击率 | 点击联系我按钮的UV占比 | ≥ 5% |
| 公众号关注转化率 | 通过资源下载引导关注的UV占比 | 上线后定义 |
| 平均停留时长 | 全站平均停留时间 | ≥ 3分钟 |
| 跳出率 | 仅访问一个页面即离开的比例 | ≤ 50% |
