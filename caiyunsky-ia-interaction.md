# 彩云译设计 · 信息架构与交互流程说明

> 基于PRD v1.0  
> 更新日期：2026-06-14

---

## 一、全局信息架构

整个站点分为五个一级页面和一个全局弹窗层，层级关系如下。

第一层（一级页面）：首页、作品集、设计方法论、设计资源、关于我。这五个页面在导航栏中平级展示，用户可以从任意页面一键跳转到其他任意一级页面。

第二层（二级页面）：作品详情页、文章详情页。它们分别从属于"作品集"和"设计方法论"，用户需要通过列表点击进入，不在导航栏直接暴露。

全局弹窗层：联系我弹窗。不占据独立页面，以模态弹窗形式覆盖在任意页面之上，从导航栏或页面内CTA按钮触发。

页面之间的从属关系：

- 首页是流量入口，不从属于任何模块，承担分发职能
- 作品集（一级） → 作品详情页（二级），一对多关系，一个分类下有多个案例
- 设计方法论（一级） → 文章详情页（二级），一对多关系，列表下有多篇文章
- 设计资源（一级）没有二级页面，下载行为通过弹窗完成
- 关于我（一级）是终端页面，没有下级页面

全局导航结构：

导航栏始终固定在页面顶部，包含以下元素——左侧：Logo或站点名称"彩云译设计"，点击回到首页；中部：作品集、方法论、资源、关于我四个导航项；右侧："联系我"按钮，视觉上高亮区分于其他导航项。移动端导航收起为汉堡菜单，"联系我"按钮在移动端以底部悬浮按钮形式常驻，不跟随汉堡菜单收起。

## 二、首页交互流程

### 页面定位

首页是整个站点的流量着陆页和分发枢纽。核心任务是在5秒内回答三个问题：这是谁、能做什么、作品水平如何。然后将访客引导到具体的作品或模块中去。

### 页面模块与层级关系

首页从上到下分为五个模块，按视觉权重递减排列。

第一模块：Hero区域。位于首屏最顶部，占据整个视口高度的60%-70%。包含三个元素：个人身份信息（名称"彩云译设计" + 一句话职业定位 + 头像或品牌图形），一张高质量视觉主图或微动效作为背景，以及一个向下滚动的引导提示。Hero区域没有可点击的交互元素，它的唯一任务是建立第一印象并引导用户向下滚动。

第二模块：能力标签导航。紧接Hero区域下方，呈现为一行水平排列的标签——UI设计、动效设计、3D设计、品牌设计。这组标签有两个作用：一是让访客快速了解能力覆盖范围，二是作为快捷入口，点击任意标签跳转到作品集页面并自动定位到对应分类tab。

第三模块：精选作品区。这是首页的核心内容区，占据最大的页面空间。以卡片网格形式展示从各分类中精选的6-8个代表性案例。每张卡片包含：作品封面图、项目名称、所属分类标签。PC端每行3-4张卡片，移动端单列排列。

第四模块：快捷入口区。在精选作品下方，以简洁的分栏形式展示三个二级模块入口——设计方法论（配一句引导语如"设计背后的思考"）、设计资源（"免费资源，关注即得"）、关于我（"了解我的设计经历"）。每个入口卡片可点击，跳转到对应一级页面。

第五模块：底部CTA区域。页面最底部、页脚之前，设置一个大面积的联系引导区域。包含一句话引导语（如"有项目想聊？"）和"联系我"按钮，点击弹出联系我弹窗。

### 用户主要操作路径

路径一（目标明确型，典型雇主行为）：着陆首页 → 看到Hero区域，了解身份 → 点击能力标签"UI设计" → 跳转到作品集页面的UI设计分类 → 逐个浏览案例。

路径二（探索浏览型，典型同行行为）：着陆首页 → 向下滚动 → 在精选作品区看到感兴趣的案例 → 点击卡片进入作品详情页 → 看完后回到首页继续浏览或通过详情页底部推荐跳转到其他案例。

路径三（快速转化型）：着陆首页 → 快速浏览后确认想联系 → 点击导航栏或底部CTA的"联系我" → 弹窗展示微信二维码和邮箱。

### 页面跳转逻辑

- 点击能力标签 → 作品集页面（对应分类tab激活）
- 点击精选作品卡片 → 作品详情页
- 点击快捷入口"设计方法论" → 设计方法论列表页
- 点击快捷入口"设计资源" → 设计资源列表页
- 点击快捷入口"关于我" → 关于我页面
- 点击"联系我"按钮 → 联系我弹窗（当前页面不跳转）
- 点击Logo → 刷新/回到首页顶部

## 三、作品详情页交互流程

### 页面定位

作品详情页是整个站点的说服力核心。访客在这里完成从"随便看看"到"这个人确实能做事"的认知转变。它需要同时服务两种阅读模式：快速扫图模式（3秒判断视觉水平）和深度阅读模式（了解项目背景和设计思考）。

### 页面模块与层级关系

整个页面是一个纵向长页面，模块从上到下排列，分为三个层级。

第一层级：项目头部信息区。包含四个元素——面包屑导航（首页 > 作品集 > UI设计 > 项目名称），项目名称（大标题），项目背景描述（选填，2-3句话说明这是什么项目、解决什么问题），我的角色说明（选填，一句话说明在项目中的职责）。项目名称是必填项，项目背景和角色说明是选填项。如果选填项为空，该区域不渲染，不留空白占位。这个区域的作用是为下方的作品图建立上下文，让雇主知道这些设计不是无源之水。

第二层级：作品展示区。这是页面的主体，占据最大的空间。作品图片以纵向瀑布流形式排列，一张接一张从上到下展开。图片之间保持统一的间距。如果案例包含视频，视频播放器穿插在图片流中的合适位置（通常放在整体展示图之后、细节图之前）。视频默认显示封面帧，点击播放，默认静音，支持全屏。在作品图片流中间或末尾，可以穿插"设计思路"文字段落（选填），用于解释特定设计决策。这种图文穿插的形式比把所有文字集中在顶部或底部更自然，阅读节奏更好。

第三层级：页面底部区域。依次包含三个模块——项目成果（选填，一小段文字说明上线效果或数据指标），CTA引导条（一句话 + "联系我"按钮，如"对这类项目感兴趣？聊聊看"），更多作品推荐（展示同分类下的2-3个其他案例卡片，形式与首页精选作品卡片一致）。

### 用户主要操作路径

路径一（快速扫图模式）：进入页面 → 扫一眼项目名称和背景 → 快速向下滚动浏览图片 → 视觉满意则点击"联系我"或点击底部推荐案例继续浏览 → 视觉不满意则点击面包屑或浏览器返回回到列表。

路径二（深度阅读模式）：进入页面 → 阅读项目背景和角色说明 → 逐张查看作品图（点击某张图放大查看细节）→ 阅读穿插的设计思路文字 → 播放视频（如有）→ 查看项目成果 → 点击"联系我"或继续浏览其他案例。

路径三（图片细节查看）：滚动浏览过程中 → 点击某张作品图 → 进入lightbox全屏查看模式 → 左右箭头切换同案例内的其他图片 → 点击关闭或按ESC回到原始滚动位置。

### 页面跳转逻辑

- 点击面包屑"作品集" → 返回作品集列表页，保持之前的分类tab和滚动位置
- 点击面包屑"UI设计"（分类名） → 返回作品集列表页并定位到该分类tab
- 点击"联系我"按钮 → 联系我弹窗（当前页面不跳转）
- 点击底部推荐案例卡片 → 进入另一个作品详情页（页面内跳转，支持浏览器回退）
- 点击lightbox内图片 → 不跳转，在当前页面以浮层展示
- 导航栏各入口 → 跳转到对应一级页面

### 状态与边界情况

- 选填字段为空时：该模块区域完全不渲染，页面自动上移补位，不出现空白区块
- 只有一张作品图时：正常展示，底部推荐区域正常显示
- 视频加载失败时：显示封面帧静态图 + "视频加载失败"提示文字
- 当前分类下无其他案例时：底部推荐区域改为展示其他分类的案例，标题改为"看看其他作品"

## 四、文章阅读页交互流程

### 页面定位

文章阅读页承担"信任加固"的角色。访客看完作品后，如果进一步阅读设计方法论文章，说明已经进入了深度了解阶段。这个页面的核心任务是提供舒适的阅读体验，并在阅读结束后自然引导转化。

### 页面模块与层级关系

页面采用经典的单栏文章阅读布局，内容区居中，最大宽度限制在720px左右以保证阅读舒适度。

第一层级：文章头部。包含四个元素——面包屑导航（首页 > 设计方法论 > 文章标题），文章标题（大标题），发布日期和阅读时长估算，封面图（可选，如果有则全宽展示）。

第二层级：文章正文区。这是页面的主体。正文以富文本形式渲染，支持以下内容格式：段落文字、小标题（h2/h3两级）、图片（居中展示，支持点击放大）、代码块（如有）、引用块、有序/无序列表。正文排版遵循以下原则：行高1.8倍，段间距适中，中文正文字号16px，确保长文阅读不疲劳。正文图片点击后进入lightbox查看，与作品详情页的图片查看行为保持一致。

第三层级：文章底部区域。正文结束后依次展示三个模块——作者卡片（头像 + 名称 + 一句话介绍 + 公众号二维码小图，引导语如"这篇文章首发于公众号「彩云译设计」"），相关文章推荐（2-3篇相关文章卡片，形式为封面缩略图 + 标题 + 摘要首行），底部CTA引导条（与作品详情页底部CTA风格一致，"想深入聊聊设计方法？" + "联系我"按钮）。

### 用户主要操作路径

路径一（完整阅读型）：从方法论列表页点击进入 → 阅读文章头部，确认是否是感兴趣的内容 → 向下滚动阅读正文 → 读完后看到作者卡片，扫码关注公众号或记下公众号名称 → 点击相关文章继续阅读，或点击"联系我"。

路径二（扫读跳出型）：进入页面 → 扫读标题和前两段 → 判断内容不感兴趣 → 点击面包屑返回方法论列表，或通过导航栏跳转到其他模块。

路径三（从搜索引擎直接着陆）：通过Google/百度搜索结果直接进入某篇文章 → 阅读正文 → 通过作者卡片了解到"彩云译设计"是一个人 → 点击导航栏"作品集"查看作品，或点击"关于我"进一步了解。

### 页面跳转逻辑

- 点击面包屑"设计方法论" → 返回方法论文章列表页
- 点击相关文章卡片 → 进入另一篇文章阅读页
- 点击作者卡片中的公众号名称或二维码 → 不跳转，二维码可长按识别（移动端）或展示放大二维码（PC端）
- 点击"联系我"按钮 → 联系我弹窗
- 导航栏各入口 → 跳转到对应一级页面

### 与公众号内容同步的交互说明

文章内容从Notion数据库读取并在站内渲染，不跳转到微信公众号原文。理由：保证PC端和移动端体验一致，避免非微信环境打开公众号链接的兼容性问题。公众号的引流通过文章底部作者卡片中的二维码自然完成。

内容同步流程对用户不可见。站长在Notion中编辑文章内容，发布后网站自动拉取渲染。文章底部注明"本文首发于公众号「彩云译设计」"作为内容来源标注。

## 五、关于我页面交互流程

### 页面定位

关于我页面是信任链条的最后一环。当访客看完作品、读完文章，还想进一步了解"这个人是谁"时才会来到这里。它的核心任务是把零散的作品印象收束为一个立体的人物认知，然后给出明确的联系方式。

### 页面模块与层级关系

页面采用纵向单栏布局，从上到下分为五个模块。

第一模块：个人形象区。位于页面顶部，包含个人照片或品牌化头像（建议用真人照片，雇主场景下真人照片比抽象头像信任感更强），名称"彩云sky"，一句话职业定位（如"视觉设计负责人 / 10年+设计经验 / 跨领域设计实践者"），以及2-3个关键身份标签。这个区域的排版可以偏品牌化，比如照片和文字的编排有一定的设计感，毕竟这是设计师的个人页面。

第二模块：个人介绍。一段自然语言的自我介绍，大约150-300字。内容覆盖：职业背景、擅长领域、设计理念、以及任何能让访客觉得"这个人有意思"的个人特质。这段文字不用写得像简历一样端着，用自然、有温度的语气说清楚自己是谁就好。

第三模块：职业经历时间线。以时间线形式展示关键职业节点，每个节点包含：时间段、公司名称、职位、以及一句话描述核心职责或代表成果。时间线纵向排列，最新经历在最上面。节点数量控制在4-6个，只放关键转折点，不需要列出每一段经历。

第四模块：能力图谱。以可视化标签或图表形式展示核心能力分布。建议用两种维度呈现——设计能力维度（UI设计、动效设计、3D设计、品牌设计、设计系统等），复合能力维度（前端开发、设计管理、内容创作、AI工具应用等）。形式可以是标签云、雷达图、或带权重的技能条，视最终视觉设计而定。这个模块的目的是让雇主快速扫一眼就知道能力边界在哪里。

第五模块：联系与社交。这是整个页面的收尾和转化终点。包含：微信二维码（大尺寸，方便扫码），邮箱地址（可点击复制），公众号"彩云译设计"二维码或链接，知识星球入口（如需要），以及一句引导语（"期待与你交流设计，扫码加我微信"）。这个区域在视觉上需要有明确的"终点感"，让访客清楚地知道：看完了，该行动了。

### 用户主要操作路径

路径一（雇主深度了解型）：从导航栏或首页快捷入口进入 → 看到个人形象区，确认身份 → 阅读个人介绍，了解背景 → 浏览职业经历时间线，确认经验匹配度 → 扫一眼能力图谱 → 到达联系区域，扫码添加微信或复制邮箱。

路径二（快速联系型）：已经看过作品，只想找联系方式 → 进入关于我页面 → 快速滚动到底部联系区域 → 扫码或复制邮箱。

路径三（搜索引擎着陆型）：通过搜索"彩云sky"或"彩云译设计"直接到达关于我页面 → 从上到下浏览了解这个人 → 通过导航栏跳转到作品集查看作品 → 满意后回到关于我页面添加联系方式。

### 页面跳转逻辑

- 点击导航栏"作品集" → 作品集列表页（这是关于我页面最可能的出方向，访客看完人想看作品）
- 点击联系区域的"联系我"或扫码 → 不跳转，在页面内直接展示二维码
- 点击公众号链接/二维码 → 不跳转，长按或扫码行为
- 点击邮箱 → 触发系统默认邮件客户端，或复制到剪贴板并toast提示"已复制"
- 导航栏其他入口 → 跳转到对应一级页面

### 特殊交互说明

关于我页面不设"联系我"弹窗入口。因为联系信息已经直接展示在页面底部，再弹窗是多余的一层。导航栏的全局"联系我"按钮在这个页面的行为改为：点击后平滑滚动到页面底部的联系区域，而非弹出弹窗。

## 六、全局交互规则

### 6.1 联系我弹窗

触发方式：点击导航栏"联系我"按钮，或点击页面内任何"联系我"CTA按钮（关于我页面除外，该页面改为锚点滚动）。

弹窗内容：半屏模态弹窗，背景遮罩。内容包含微信个人二维码（居中，大尺寸）、二维码下方的引导语、邮箱地址（可点击复制）、关闭按钮（右上角X）。点击遮罩区域或按ESC也可关闭。

弹窗不打断当前页面状态，关闭后用户回到之前的浏览位置。

### 6.2 页面切换与状态保持

从作品详情页返回作品集列表页时，列表页保持之前的分类tab选中状态和滚动位置，不重置到顶部。

从文章详情页返回方法论列表页时，同理保持滚动位置。

一级页面之间的跳转不保持滚动状态，统一回到页面顶部。

### 6.3 加载与过渡

页面之间的跳转使用平滑的淡入过渡，不使用硬切。

图片全部采用懒加载，进入视口前显示骨架屏占位（灰色色块，保持图片实际比例）。

长页面（作品详情页、文章阅读页）在首屏内容加载完成后即可交互，后续图片异步加载不阻塞。

### 6.4 移动端适配要点

导航栏收为汉堡菜单，"联系我"按钮脱离菜单，以底部悬浮按钮形式常驻。

所有弹窗在移动端改为底部抽屉式上滑面板，而非居中模态。

图片lightbox在移动端支持手势缩放和左右滑动切换。

作品卡片网格在移动端改为单列，卡片宽度占满屏幕。
